## Task

Now, you will carefully consider all staged changes in the repo at `{project-repo-location}`, and you will wisely create a set of git commits.

Analyze every staged change. Group related changes into logically coherent commits — each commit should represent one meaningful unit of work, using standard granularity. Every staged change must appear in exactly one commit. No changes may be silently dropped.

The creative part is purely in the commit message text: every message must be written in iambic pentameter. The messages must still be accurate and descriptive of the actual changes — do not sacrifice clarity for meter.

## Commit Message Style

Every commit message must be a line of iambic pentameter (da-DUM da-DUM da-DUM da-DUM da-DUM — ten syllables, five iambic feet). The message must still accurately describe the change. Lowercase letter leading. 90 characters maximum. No message body — subject line only.

### Example

```
bestow upon the course a fairer name
remove the null that haunts the student's call
refactor queries burdened by their weight
enhance the guard that shields enrollment's gate
append the field the campus form requires
```
